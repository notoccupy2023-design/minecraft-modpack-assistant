# 我的世界整合包助手

把整合包目录整理成玩家拿来就能用的中文指南，不输出原始 Mod 清单，也不臆造玩法。

## 默认执行流程

1. 先读取项目内的 `AGENTS.md`，再检查 `mods/`、`options.txt`、`config/`、`defaultconfigs/`、任务配置、现有生成脚本和 HTML。
2. 用户只给一个路径时直接执行，不追问输出格式。路径是 `mods/` 时使用其父目录；未给路径时从当前目录查找。
3. 运行本 Skill 目录中的 `scripts/modpack_scan.py <整合包目录> --report` 获取 Mod 清单和可运行性报告。首次使用或修改脚本后运行 `--self-check`。
4. 从 `manifest.json`、启动器实例配置和 jar 元数据判断 Minecraft 版本及 Forge/NeoForge/Fabric/Quilt 环境。加载器与下载来源分开记录。
5. 优先读取 jar 内的 `META-INF/neoforge.mods.toml`、`META-INF/mods.toml`、`fabric.mod.json` 或 `quilt.mod.json`，文件名只作后备。
6. 按“资料查询规则”查询每个 Mod；优先使用 Modrinth 精确版本信息，其次查 CurseForge，再用 BBSMC 补充中文资料。重复扫描时复用项目内已验证内容，只补新增或变化的 Mod。
7. 默认生成一个 UTF-8、CSS/JS 内联、可离线打开的 HTML。若项目已有生成器，只改生成器并重新生成成品。
8. 完成后先报告可运行性阻断和已知冲突，再报告玩法风险与按键提示；最后检查 Mod 数量、搜索、脚本语法、中文编码和前十张卡片排序。

## 即开即用默认值

- 默认输出：整合包根目录下的 `modpack-guide.html`。
- 默认语言：简体中文；英文资料翻译为中文，Mod 名、命令、ID、键名和版本号保持原样。
- 默认首页顺序：环境与风险、开始游玩、核心玩法、操作手册、Mod 索引、资料来源。
- 默认优先回答：能否正常启动、明确不兼容项、玩法重叠风险、任务/指南入口和核心操作。
- 缺少可靠资料时保留本地事实并标记“待核实”，不要补写猜测。
- 不生成安装器、不修改 jar、不改玩家配置，除非用户明确要求。

## 资料查询规则

- 从 jar 文件名移除 Minecraft 版本、Mod 版本、加载器、客户端/服务端及 alpha/beta/release 等后缀。例如 `Argentina's delight 1.20.1 (3.0 beta).jar` 取 `Argentina's delight`。
- 先用扫描结果中的 SHA-1 调用 Modrinth `version_file` 查询精确版本；这是首选分发平台。
- Modrinth 哈希查询无结果时，使用清洗名搜索 Mod，并按 `project_type=mod`、Minecraft 版本和加载器筛选；核对名称或 modId 后取第一个可靠结果。
- Modrinth 无可靠结果时，先采用 jar 元数据中的精确项目主页；仍未定位时再搜索 CurseForge 的 Mod 分类。CurseForge 存在多个结果时默认取第一个，但必须核对游戏版本和加载器。
- BBSMC 作为中文社区补充来源：可用于核对中文名称、补充中文简介、整合包改动、更新日志、讨论和反馈线索；必须核对项目、Minecraft 版本与加载器，不得覆盖 Modrinth、CurseForge、作者主页或 jar 元数据中的精确事实。
- 最后再查 GitHub README、官方 Wiki、作者 issue tracker 或其他论坛；记录实际使用的页面链接和对应版本。
- 每个 Mod 独立介绍，尽量保留原页面的标题和段落顺序，不把 Mod 索引写成整合包攻略。
- 提取原页面实际提供的简介、重要物品/方块/实体、按键、控制、前置条件和交互逻辑。
- 基础机制只在基础 Mod 中说明一次；附属卡片只写自己的新增内容。库/API 简写为“无需操作；后台前置”。
- 页面内容与当前 Minecraft/加载器版本不符时，不把它当作当前环境事实。

## 整合包冲突分析

- **可运行性阻断**：检查加载器/游戏版本不匹配、重复 modId、缺失必需前置、Mod 元数据声明的 `breaks`/`conflicts`/`incompatible`，以及 Modrinth 版本依赖中的 `incompatible`。
- **实际故障**：若存在 `logs/latest.log` 或 `crash-reports/`，优先依据首个根因、加载失败信息和涉及 modId；日志证据高于静态推测。
- **已知冲突**：优先查当前 Modrinth 项目与精确版本，再查作者 issue tracker 和 CurseForge；BBSMC 的讨论与反馈用于发现线索。必须记录冲突双方、受影响版本、现象、规避方式和来源，社区线索未经版本匹配或本地复现不得标成已确认。
- **玩法冲突**：结合 Mod 描述与本地 `config/`、任务、KubeJS/CraftTweaker、配方脚本，检查重复地图/配方查看器/背包/墓碑/连锁挖掘/库存整理，以及世界生成、经济、难度和进度被多个 Mod 同时改写。
- 不对所有 Mod 做无依据的两两组合搜索。先按功能分组，只复核重叠或高风险组。
- 使用“已确认”“高风险”“需关注”“提示”分级；没有版本匹配或来源证据时不得写成已知冲突。

## 快捷键规则

- 完整读取客户端 `options.txt` 的 `key_...:key.keyboard...` 或鼠标绑定；不要把服务端配置当成真实客户端按键。
- 优先使用 jar 内 `zh_cn.json`/`zh_cn.lang` 的声明名，再用 `en_us.json`/`en_us.lang` 补缺。
- 没有真实绑定时同时显示 `未绑定` 和“Mod 声明的快捷键名称”。
- 同一真实按键绑定多个功能时标记为提示；除非日志显示它影响操作，否则排在可运行性和玩法风险之后。
- 对任务书、地图、背包、连锁挖掘、模式切换等高频功能优先展示绑定状态。

## 内容和排序

- 推荐路线只描述整合包自身的前期、中期、后期目标；不要复述所有 Mod 卡片。
- 操作手册写玩家立刻需要的入口和步骤，优先引用任务配置、本地配方和真实绑定。
- Mod 索引按以下顺序：主题核心与 JEI/REI/Jade 类工具、内容附属、自动化/存储/探索、装饰与体验、性能、库/API。
- 搜索字段只包含名称、modId、jar 名、分类和短简介，避免通用建议造成误命中。
- 每张卡片至少包含中文标题、原名/modId、简述、关键内容、操作/交互、按键状态、jar 名和来源链接；有版本化证据时增加兼容性/已知问题，没有证据的字段省略。

## 安全与验收

- 不删除、移动或修改 `mods/` 内文件；禁止批量删除任何文件或目录。
- 保留用户已有成果，生成 HTML 前优先更新源生成器。
- 验收时确认：UTF-8 meta 存在；唯一 Mod 数与卡片数一致；阻断问题位于报告顶部；每条已知冲突有版本和来源；玩法推测明确标成风险；内联 JS 可解析；游戏内容排在库/API 前。

实现或修改 HTML 结构、来源显示、排序和卡片内容时，读取 `references/guide-pattern.md`。
